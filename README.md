## Design and Implementation of LangChain Expression Language (LCEL) Expressions

### AIM:
To design and implement a LangChain Expression Language (LCEL) expression that utilizes at least two prompt parameters and three key components (prompt, model, and output parser), and to evaluate its functionality by analyzing relevant examples of its application in real-world scenarios.

### PROBLEM STATEMENT:
LangChain Expression Language (LCEL) enables efficient interaction with Large Language Models (LLMs) by allowing developers to build modular and reusable pipelines. In this task, the objective is to design an LCEL pipeline that can generate structured responses based on dynamic inputs.

The process involves creating a prompt that accepts parameters such as topic and length, connecting it with a language model, and using an output parser to convert the model’s response into a structured format. The system demonstrates how LCEL components work together to produce meaningful and organized outputs.
   
### DESIGN STEPS:
# Step 1: Create a Structured Output Model
Define a Pydantic model called SummaryResponse to specify the structure of the expected output.
Attach a PydanticOutputParser to ensure the generated response follows the defined schema.

# Step 2: Design the Prompt Template
Use ChatPromptTemplate to build a prompt that accepts parameters such as topic, tone, audience, and length.
Include escaped formatting instructions to ensure the response is generated in valid JSON format.

# Step 3: Configure the Language Model
Initialize the ChatGroq language model and configure settings such as model name, temperature, and API key for response generation.

# Step 4: Build and Execute the LCEL Chain
Combine the prompt, model, and parser using the LCEL pipe operator (|).
Execute the chain using .invoke() with sample input parameters and observe the validated structured output generated by the system.
```py
# RAGA SUSANTH
# 212224230217
# INSTALLS:
# pip install -U langchain-core langchain-groq pydantic python-dotenv

from pydantic import BaseModel, Field
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import PydanticOutputParser
from langchain_groq import ChatGroq
import os

os.environ["GROQ_API_KEY"] = "groq_api"

# 1. STRUCTURED MODEL
class SummaryResponse(BaseModel):
    summary: str
    word_count: int
    highlights: list[str]

parser = PydanticOutputParser(pydantic_object=SummaryResponse)

# ESCAPE THE FORMAT INSTRUCTIONS
format_rules = parser.get_format_instructions().replace("{", "{{").replace("}", "}}")

# 2. PROMPT
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "You are a concise factual summarizer."),
        ("human",
         "Write a {length}-word {tone} summary about: {topic}.\n"
         "Provide exactly 3 short highlights.\n\n"
         "Output MUST be valid JSON using this schema:\n"
         f"{format_rules}\n\n"
         "Audience: {audience}"
        ),
    ]
)


# 3. MODEL (GROQ)
llm = ChatGroq(
    model="openai/gpt-oss-20b",
    temperature=0
)

# 4. LCEL CHAIN
chain = prompt | llm | parser

# 5. TEST EXAMPLES
examples = [
    {"topic": "Climate change causes", "length": "50", "tone": "neutral", "audience": "general readers"},
    {"topic": "Transformer neural networks", "length": "40", "tone": "technical", "audience": "ML engineers"},
]

for ex in examples:
    print("\n=== INPUT ===")
    print(ex)

    result = chain.invoke(ex)
    print("\n=== PARSED JSON OUTPUT ===")
    print(result.model_dump())

    real_wc = len(result.summary.split())
    print("Reported:", result.word_count)
    print("Actual:", real_wc)
```
